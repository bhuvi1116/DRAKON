# Stage 27 — DRAKON Layout Engine

На этом шаге редактор получает отдельный слой авто-компоновки для visual-first профиля ДРАКОН.

Что добавлено:
- `DrakonLayoutEngine`
- `DrakonLayoutOptions`
- `DrakonLayoutReport`
- `DrakonLayoutIssue`

Цели:
- выравнивание основных икон по шампуру;
- поддержка силуэтной раскладки с правыми ветвями;
- контроль правила "чем правее — тем позже";
- подготовка к следующему шагу, где layout будет подключен к canvas и применяться интерактивно.

Текущее поведение:
- `Title`, `Start`, `Question`, `Condition` и линейные иконы выравниваются по шампуру;
- `BranchStart` и `Address` могут быть вынесены в правую ветвь через `Lane > 0`;
- отрицательные lane для силуэта считаются нарушением visual-профиля;
- layout возвращает диагностический отчет, который можно будет показать в UI.

Ограничения:
- на этом шаге layout engine еще не подключен к кнопке в редакторе;
- нет автоматической трассировки линий после перекомпоновки;
- правило шампура пока задается координатно, а не через полноценную визуальную ось.
