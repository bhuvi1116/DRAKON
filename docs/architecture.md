# Архитектура DRAKON-NX

## Решение по UI

Редактор строится на Avalonia UI, потому что это единственный практический способ сохранить XAML/MVVM-подход и получить реальную кроссплатформенность.

## Слои

- Core: диаграмма, узлы, связи, базовые инварианты.
- Validation: правила корректности и диагностика.
- Serialization: JSON DTO и маппинг.
- Editor: UI и orchestration.

## Базовый pipeline

1. Пользователь создает диаграмму.
2. Диаграмма валидируется.
3. Корректная диаграмма позже будет преобразована в IR.
4. IR позже будет превращаться в C.
