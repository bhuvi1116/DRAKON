# Адаптация редактора под визуальную спецификацию ДРАКОН

Этот шаг переводит редактор в visual-first режим и опирается на базовые правила из приложенной спецификации:

- различаются два режима: `primitive` и `silhouette`;
- для силуэта важны иконы `заголовок`, `имя ветки`, `адрес`, `конец`;
- вопрос и развилка остаются центральным визуальным примитивом;
- для силуэта вводится пространственное правило `чем правее - тем позже`;
- главный маршрут должен оставаться на шампуре ветки;
- редактор начинает различать исполнимый профиль `ExecutableV0` и визуальный профиль `DrakonVisualSpec`.

## Что изменено в модели

- `DiagramDocument.Profile`
- `DiagramDocument.LayoutMode`
- расширенный `NodeKind` для ключевых икон ДРАКОН
- `DiagramNode.Lane` для визуального размещения веток

## Что изменено в редакторе

- по умолчанию открывается силуэтный sample
- canvas рисует разные формы для `заголовка`, `имени ветки`, `вопроса`, `адреса`, `конца`
- для силуэта добавлены направляющие веток
- в UI показан текущий профиль и режим схемы

## Текущее ограничение

На этом шаге visual profile не пытается генерировать C-код.
Кодогенерация сохранена только для старого исполнимого подмножества.

Следующий шаг должен усилить canvas-редактор именно под ДРАКОН:

- порты икон на самой диаграмме;
- явные точки подключения для вопроса и адреса;
- визуальная проверка шампура и правила `чем правее - тем позже`.
