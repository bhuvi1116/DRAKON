# Command-based history v0

На этом этапе редактор переходит от чистого snapshot-подхода к гибридной модели истории:

- крупные операции уровня документа могут восстанавливаться через snapshot;
- операции редактирования графа выполняются через явные undoable actions;
- история хранит стек действий и отдельно стек redo.

## Поддержанные действия

- добавление узла;
- удаление узла;
- изменение текста узла;
- добавление связи;
- удаление связи;
- переименование документа через snapshot-action.

## Ограничения

Пока используется легковесная delegate-based команда. Следующий шаг — выделить отдельные типы команд для координат, свойств узлов и перемещения на canvas.
