# Сборочный пайплайн v0

На шаге 5 проект получает отдельный слой `Build`, который отвечает за экспорт generated C-кода и оркестрацию внешней сборки.

## Последовательность

1. Валидация диаграммы.
2. Генерация `main.c` через `CodeGen`.
3. Генерация `CMakeLists.txt`.
4. Экспорт файлов в отдельный каталог.
5. Вызов `cmake -S <dir> -B <dir>/build`.
6. Вызов `cmake --build <dir>/build --config Release`.
7. Поиск готового бинарника.
8. Запуск бинарника и захват stdout/stderr.

## Структура каталога экспорта

```text
<export-dir>/
  main.c
  CMakeLists.txt
  build/
    ...
```

## Ограничения текущей версии

- UI пока показывает подготовленные артефакты и путь экспорта, но еще не содержит интерактивных кнопок «Собрать» и «Запустить».
- Внешняя сборка реализована сервисом `CMakeBuildService` и готова к подключению в следующем шаге.
- Предполагается наличие системного `cmake` и C-компилятора в PATH.
