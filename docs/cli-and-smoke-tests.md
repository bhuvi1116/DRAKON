# CLI и smoke-проверки

На этом шаге в проект добавлен отдельный консольный инструмент `drakon-nx`.

Он нужен для двух задач:

1. Прогонять проверки и кодогенерацию без UI.
2. Давать стабильную основу для CI/CD и автоматических smoke-проверок.

## Команды

- `validate <diagram.json>` — проверить диаграмму
- `generate <diagram.json> <output-dir>` — экспортировать `main.c` и `CMakeLists.txt`
- `build <diagram.json> <output-dir>` — экспортировать проект, запустить `cmake`, собрать и выполнить бинарник

## Почему это добавлено сейчас

После появления UI-команд сборки важен второй интерфейс системы: не визуальный, а скриптуемый.
Это упрощает:

- интеграцию с CI;
- пакетную проверку sample-программ;
- локальную диагностику без запуска desktop-редактора;
- подготовку будущих integration и smoke tests.

## Базовый сценарий

1. Проверить sample-файл через `validate`.
2. Сгенерировать проект через `generate`.
3. Запустить `build` и убедиться, что бинарник собрался и выполнился.

## Ограничение текущего шага

В текущей среде контейнера недоступен .NET SDK, поэтому фактический прогон `dotnet build`, `dotnet test` и `dotnet run` не подтвержден здесь инструментально. Но структура проекта, команды и тесты подготовлены под реальный .NET 8 workflow.
